# 1644 — IR hygiene full-pipeline observability (refine #1047)

**Status:** shipped (close pending)

## Context

`SyntaxMarker::MacroIntroduced` is the cornerstone of Aura's macro hygiene
contract — a `query:macro!` invocation that uses `clone_macro_body` stamps
all introduced nodes with `MacroIntroduced`, and downstream passes skip /
deopt for any `MacroIntroduced` code unless they explicitly opt in via
`InlinePass::set_respect_macro_hygiene(false)`.

The AST-layer hygiene was tightened over multiple predecessors (#1047,
#455, #246, #1610, #1616, #1273, #757). What remained was a thin IR /
observability layer where (a) the `query:ir-marker-stats` primitive
could not iterate `IRModule.instructions` from C++-side (it iterated the
AST `marker_column` instead — comment in the impl said "IR per-instruction
markers are a follow-up when IRModule is always reachable"), and (b)
the runtime had no per-CompilerMetrics counters for the cross-marker
inliner-skip path or the lowering source_marker propagation path.

## What landed

### 1. Two new `CompilerMetrics` counters (AC4)

```cpp
// src/compiler/observability_metrics.h
std::atomic<std::uint64_t> ir_macro_introduced_inlined_skipped_total{0};
std::atomic<std::uint64_t> lowering_marker_propagated_total{0};
```

The corresponding `X-macro` fields were added in
`src/compiler/compiler_metrics_fields.inc` so the C-side counters
exposed via `aura_*_read()` keep in sync without code drift.

### 2. Paired legacy / new bumps (5 wire-up sites, 4 unique)

| File                                    | Site                                                | Counter bumped                                       |
|-----------------------------------------|-----------------------------------------------------|-----------------------------------------------------|
| `src/compiler/pass_manager.ixx`         | outer `InlinePass::run_on_block` skip               | `ir_macro_introduced_inlined_skipped_total`         |
| `src/compiler/pass_manager.ixx`         | `is_inlinable_branch_aware` hygiene gate (2 cases)  | `ir_macro_introduced_inlined_skipped_total` (×2)    |
| `src/compiler/lowering.ixx`             | AoS `source_marker` assignment                     | `lowering_marker_propagated_total`                  |
| `src/compiler/lowering.ixx`             | SoA dual-emit `source_marker` mirror               | `lowering_marker_propagated_total`                  |

All four (counting the two `is_inlinable_branch_aware` cases) sites are
paired with the **legacy** namespace-scope counters (`macro_hygiene_skipped_`
in `pass_manager.ixx`; `hygiene_ir_macro_marker_total` + the C-shared
`aura_hygiene_ir_macro_marker_inc()` in `lowering.ixx`) so existing
tests + dashboards keep their semantics unchanged. The new
`ev->bump_*()` call honors the `Evaluator::yield_hook_evaluator()` null
fallback (per the #1908 / #1641 module-boundary pattern) so calls from
TUs without the Evaluator module safely no-op.

### 3. `(query:ir-marker-stats)` now iterates `IRModule.instructions` (AC3)

The primitive's implementation in `evaluator_primitives_query.cpp`
extended to:

- Walk `CompilerService::last_ir_module()->functions[*].blocks[*]
  .instructions[*].source_marker` (AC3 — the authoritative IR-layer
  surface).
- Read the two new `CompilerMetrics` counters for closed-loop
  observability (AC4 — composition into the same primitive's hash).
- Keep the existing per-call counter bump on
  `ir_marker_stats_queries_total` (back-compat) so callers tracking the
  query rate don't drift.

The returned FlatHashTable now has **5 fields** instead of 3:

```
{
  "ir_user": int,
  "ir_macro_intro": int,
  "ir_bool_lit": int,
  "lowering_marker_propagated": int,           // Issue #1644
  "ir_macro_introduced_inlined_skipped": int,  // Issue #1644
}
```

This is non-breaking — older callers that destructured only the first
3 fields continue to work; new fields are additive observability.
