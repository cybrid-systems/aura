# M4 Domain (`m4-*`) — Issue #1976

**Status: deferred linear-type stub surface (SlimSurface domain prefix).**

Parent: #1965 (Phase 3 commercial_readiness scope) · Decision: **KEEP**.

## Decision

The 3 `m4-*` primitives are **kept** under a budget as Issue #108 M4
linear-type stubs, not deleted. They are already excluded from SlimSurface
*core* via `DOMAIN_STATUS["m4-"] = "deferred"`.

| Option | Chosen | Why |
|---|---|---|
| Remove | No | Regression suites (`06_linear.aura`, `07_borrow_checker.aura`) and fixtures reference the names. |
| Keep + gate | **Yes** | Completes the #1965 deferred-domain follow-up series (#1967–#1976). |

**Naming note:** The issue title said “m4 macro processor,” but the
implementation is **linear ownership stubs** (`m4-move` / `m4-borrow` /
`m4-return!`) from Issue #108 part 3 — not GNU m4. The domain prefix
`m4-` is historical.

## Build flag: `AURA_ENABLE_M4`

CMake option (default **ON**):

```bash
cmake -B build -S .
cmake -B build_slim -S . -DAURA_ENABLE_M4=OFF
```

When OFF, the three `m4-*` adds in
`register_math_regex_and_arithmetic_primitives`
(`evaluator_primitives_math.cpp`) are not registered.

**Not gated:** `define-linear`, regex primitives, arithmetic (`+`, `-`, …).
