## 🟢 Quick Wins (1-2 天，高影响低风险)

### 1. 去除 LoweringPass 类的可变状态

**当前**：`LoweringPass` 是一个 class，内部有 `scopes_`、`free_var_map_` 等可变成员。

**目标**：用纯函数 + `LoweringContext` 值结构体替代。

```
// 当前
LoweringPass lp(arena);
lp.lower_lambda_body(...)  // 修改内部状态

// 目标
LoweringContext ctx{arena, ...};
auto result = lower_lambda_body(ctx, ...);  // ctx 是值语义
```

**文件**：`lowering.ixx` + `lowering_impl.cpp`

### 2. collect_free_vars 返回结果而非 out-param

**当前**：
```cpp
void collect_free_vars(const Expr*, unordered_set<string>& free, unordered_set<string>& bound);
```

**目标**：
```cpp
auto collect_free_vars(const Expr*) -> std::pair<unordered_set<string>, unordered_set<string>>;
```

### 3. NodeTag switch 加 consteval 验证

**当前**：`kNodeMeta` 表手动维护数组大小（`std::array<NodeMeta, 11>`）。

**目标**：
```cpp
consteval bool is_valid_tag(NodeTag t) { return meta(t).name != "unknown"; }
consteval auto max_tag() { return NodeTag::Quote; }  // 编译期边界
```

### 4. refs_of/match 改用 ranges（已完成部分）

从 P1-P4 优化中已开始（`by_tag()` 返回 span，`calls_to` 用 filter）。
进一步：match 递归 → 迭代器 range。

## 🟡 Medium (3-5 天，架构级改进)

### 5. FlatAST 完全替换指针 Expr

**当前**：两个 AST 共存——`Expr*` (variant tree) + `FlatAST` (SoA)。Parser 产出 FlatAST，但 `CompilerService::eval()` 用 `reconstruct_node()` 转回 Expr*。

**目标**：tree-walker 直接消费 FlatAST。消除 `reconstruct_node` 桥接。

**依赖**：需要 tree-walker evaluator 适配 FlatAST 而非 Expr*。

**路线**：
1. 在 `Frontend`（tree-walker evaluator）上加一个 `eval_flat(FlatAST&, NodeId root)` 方法
2. 逐步替代 `eval(Expr*)`

### 6. Lowering 纯函数管道

**当前**：`lower_expr()` 是大递归函数。

**目标**：分解为小纯函数管道。

```
FlatAST → lower_to_flat(...) → IRBuilder → IRModule
             ↑                    ↑
         纯函数                  builder 值类型
```

### 7. SymId 完全替代 String（节点内）

**当前**：部分节点用 `std::string`（`VariableNode::name`，`DefineNode::name`），部分用 `SymId`（FlatAST 版本）。Expr* 路径还在用 string。

**目标**：Expr* 节点也使用 `SymId` + `StringPool` 引用。

## 🔴 Major (1-2 周，结构性变更)

### 8. Contracts + 边界检查

**当前**：无 `[[assert:]]` / `[[expects]]` / `[[ensures]]`。

**目标**：在 FlatAST 和 IR 的热路径加入 contracts。

```cpp
NodeView get(NodeId id) const
    [[expects: id < size()]]
    [[ensures: ret.tag != NodeTag::LiteralInt || ret.has_int()]];
```

**注意**：GCC 16.1 的 contract 支持状态？需要验证 `-fcontracts` 可用性。

### 9. SIMD 向量化分析 Pass

**当前**：`const_fold`、`compute_kind`、`arity` 三个 pass 逐节点串行。

**目标**：对 SoA 数组用 SIMD（`std::experimental::simd` 或手工向量化）。

```
// 当前：for each node: switch(tag) → process
// 目标：auto tags = span(tag_.data(), size());
//       按 tag 分组 → 批量处理同类型节点
```

### 10. 查询引擎进化

**当前**：QueryEngine 用 S-表达式解析 → match() 递归。

**目标**：QueryExpr → 编译为 range pipeline（如把 `(and (node-type Call) (callee "+"))` 编译为 `by_tag(Call) | filter(callee_is("+"))`）。

**已在 P1-P4 优化中部分实现**（TagIndex + SymRefIndex）。
下一步：查询编译（P2996 JIT）。
