# Aura 已知问题

> 持续跟踪的已知 bug 和设计限制。

## 🔴  闭包按值捕获环境快照

`define` 创建闭包时捕获当前 env 的深拷贝。后续定义的函数在闭包中不可见。

**影响**：
- 互递归函数需要在同一 `begin` 内才能互相可见（已用 letrec 语义修复）
- json.aura 已随 letrec 修复简化（移除了 `_STR` hack）

**修复状态**：✅ 内部 define 已改用 letrec 两阶段求值（2026.05.16）

## 🔴  `"` 在字符串中不支持转义

Aura 的 lexer 不支持 `\"` 作为字符串中的引号转义。反斜杠直接按字面量处理。
这意味着在字符串中包含 `"` 字符的唯一方式是运行时构造 (`integer->char 34`)。

**影响**：
- json.aura 测试需要 runtime 构造 JSON 字符串
- 设计文档和 README 中的示例要避免内联 `"`

**修复状态**：⬜ 待做（lexer 字符串解析需转义处理）

## 🟡  defmacro 支持 `(name . args)` 点列表

当前 `defmacro` 参数列表不支持 `(name . args)` 这种点列表变参形式。
变参宏需要通过传列表参数 + `, @fields` 变通。

**修复状态**：⬜ 待做（parser 层支持点列表参数）
